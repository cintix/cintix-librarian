# Configuration Reference

## Project Configuration (`librarian.json`)

### Top-Level Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `libDir` | String | `"lib"` | Directory for downloaded JARs, relative to project root |
| `defaultRepository` | String | `"maven-central"` | Default repository name for dependencies |
| `repositories` | Object | Maven Central | Repository definitions |
| `dependencies` | Object | (empty) | Dependency declarations |

### `repositories`

```json
{
  "repositories": {
    "default": "maven-central",
    "items": {
      "maven-central": {
        "type": "maven",
        "url": "https://repo1.maven.org/maven2"
      },
      "company-releases": {
        "type": "maven",
        "url": "https://repo.company.local/repository/releases"
      }
    }
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `default` | String | Name of default repository (must exist in items) |
| `items` | Object | Map of repository name â†’ definition |

Repository definition:

| Field | Type | Description |
|-------|------|-------------|
| `type` | String | Always `"maven"` |
| `url` | String | Base URL of Maven repository |

### `dependencies`

Each key is a Maven coordinate (`groupId:artifactId`). The value can be:

**Simple form** (version string only):

```json
"org.slf4j:slf4j-api": "2.0.16"
```

**Detail form** (object):

```json
"com.fasterxml.jackson.core:jackson-databind": {
  "version": "^2.17",
  "transitive": true,
  "repository": "company-releases"
}
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `version` | String | (required) | Version constraint |
| `transitive` | Boolean | `false` | Whether to fetch transitive dependencies |
| `repository` | String | (default repo) | Named repository to use |

### Version Constraints

| Syntax | Type | Example | Resolution |
|--------|------|---------|------------|
| `X.Y.Z` | Exact | `2.0.16` | Uses exact version |
| `X.Y.*` | Patch stream | `2.1.*` | Highest Z in X.Y |
| `X.*` | Major stream | `2.*` | Highest Y.Z in X |
| `^X.Y` | Compatible range | `^2.17` | Highest >= X.Y.0, < (X+1).0.0 |
| `latest` | Latest stable | `latest` | Highest non-prerelease version |

## Lock File (`librarian.lock.json`)

Generated by `librarian sync` or `librarian resolve`. Contains exact resolved versions.

```json
{
  "version": "1",
  "dependencies": {
    "org.slf4j:slf4j-api": {
      "requested": "2.0.16",
      "resolved": "2.0.16",
      "repository": "maven-central",
      "checksum": "0172931663a09a1fa515567af5fbef00897d3c04"
    },
    "com.google.code.gson:gson": {
      "requested": "^2.11",
      "resolved": "2.14.0",
      "repository": "maven-central",
      "checksum": "efc0e34ede4e3204eaefb84a00e55e8c86634382"
    }
  }
}
```

| Field | Description |
|-------|-------------|
| `version` | Lock file format version (currently `"1"`) |
| `dependencies` | Map of coordinate â†’ resolved entry |
| `requested` | Original version constraint from librarian.json |
| `resolved` | Exact resolved version |
| `repository` | Repository used for resolution |
| `checksum` | SHA-1 hash of downloaded JAR (null if resolve-only) |

## Global Configuration (`~/.librarian/config.json`)

User-wide defaults. Same structure as librarian.json, but only fields that differ from defaults need to be specified.

```json
{
  "libDir": "external/lib"
}
```

Project configuration takes precedence over global configuration. If neither specifies a value, the built-in default is used.

## Defaults Summary

| Setting | Default |
|---------|---------|
| `libDir` | `"lib"` |
| `defaultRepository` | `"maven-central"` |
| Maven Central URL | `https://repo1.maven.org/maven2` |
| `transitive` | `false` |
| Cache | disabled |
